🚀 แผนการอัปเกรด Bait Team Check-in

สรุปขั้นตอนย้ายโครงสร้างสู่ Next.js และเพิ่มระบบคำนวณระยะทาง Google Maps

📦 1. เปลี่ยนโครงสร้างโปรเจกต์ (Next.js Migration)

เป้าหมายคือการย้ายจาก Vanilla HTML/JS ไปเป็นระบบที่มี Frontend และ Backend ในตัว เพื่อความปลอดภัยในการเก็บ API Key

  • โครงสร้างใหม่: ใช้ Next.js (App Router)
  • Frontend: เปลี่ยนไฟล์ .html (เช่น index.html, mobile-checkin.html) เป็น React Components (page.tsx)
  • Backend (API Routes): สร้างโฟลเดอร์ app/api/ เพื่อซ่อนโค้ดที่ติดต่อกับ Firebase Admin และ Google Maps API
  • Styling: เปลี่ยนไปใช้ Tailwind CSS เพื่อให้จัดหน้าเว็บได้ไวขึ้นและเป็นระเบียบ

🗺️ 2. การเชื่อมต่อ Google Maps API Key

สิ่งที่ต้องทำใน Google Cloud Console:

  1. สร้างโปรเจกต์ใหม่ใน Google Cloud Console
  2. เปิดใช้งาน API: Distance Matrix API และ Geocoding API
  3. สร้าง API Key (Credentials)
  4. การรักษาความปลอดภัย (สำคัญมาก):
    • เก็บ API Key ไว้ในไฟล์ .env.local ของ Next.js (ห้ามเอาขึ้น GitHub)
    • เรียกใช้ API Key ผ่านฝั่ง Server (API Routes) เท่านั้น ห้ามเรียกจากฝั่ง Client เด็ดขาด

💰 3. การตรวจสอบโควต้าและการป้องกันบิลช็อก

📊 วิธีเช็คโควต้าใช้งาน (Usage)

  1. เข้า Google Cloud Console
  2. เมนูซ้ายเลือก Google Maps Platform
  3. คลิกที่ Quotas หรือ Metrics
  4. ดูจำนวนครั้งที่ API ถูกเรียกใช้ในเดือนปัจจุบัน

🚨 วิธีตั้งเตือนงบประมาณ (กันเงินไหล)

  1. เมนูซ้ายเลือก Billing -> Budgets & alerts
  2. คลิก Create budget
  3. ตั้งชื่อ (เช่น Maps Budget) และเลือกโปรเจกต์
  4. ตั้ง Target Amount ไว้ที่ $1.00
  5. ให้ระบบส่ง Email เตือนเมื่อยอดถึง 50% และ 100% ของ $1

🤖 4. Prompts สำหรับ Agentic IDE (Windsurf)

แนะนำให้ก็อปปี้คำสั่งให้ Windsurf ทำทีละ Phase เพื่อป้องกัน AI สับสนและเกิดบั๊กครับ (Prompt เขียนเป็นภาษาอังกฤษเพื่อให้ AI เข้าใจบริบทการเขียนโค้ดได้ดีที่สุด)

Phase 1: Setup Next.js & UI Migration

I want to migrate my current vanilla HTML/JS project "Bait Team Check-in" to Next.js (App Router) using Tailwind CSS.

1. Initialize a new Next.js project in this directory.
2. Convert the existing `index.html` (Admin Dashboard) and `mobile-checkin.html` into Next.js React components (`app/page.tsx` and `app/checkin/page.tsx`).
3. Maintain the exact same UI/UX but rewrite the styling using Tailwind CSS utility classes.
4. Do not implement Firebase logic yet, just setup the UI structure and mock data.

Phase 2: Firebase Integration (Secure Backend)

Now, let's connect Firebase securely.

1. Set up Firebase Client SDK for the frontend components.
2. Set up Firebase Admin SDK. Create a secure Next.js API Route (`app/api/checkin/route.ts`).
3. When a user submits a check-in from `app/checkin/page.tsx`, it should send a POST request to our API Route.
4. The API Route should use Firebase Admin to save the check-in data (team ID, timestamp, lat, lng) securely to Firestore.

Phase 3: Google Maps Distance API Logic

I need to calculate the distance traveled between check-ins using Google Maps Distance Matrix API.

1. I will provide my GOOGLE_MAPS_API_KEY in the `.env.local` file.
2. Update the `app/api/checkin/route.ts`. Before saving the new check-in to Firestore, fetch the user's LAST check-in location from Firestore.
3. If a previous location exists for today, call the Google Maps Distance Matrix API from the server-side to calculate the distance (in km) and travel time (in mins) between the last location and the current location.
4. Save this calculated `distanceFromLastPoint` and `travelTimeFromLastPoint` into the new Firestore check-in document.

Phase 4: Workload Dashboard Calculation

Finally, let's build the Workload Dashboard for the Admin.

1. In `app/page.tsx`, fetch today's and this month's check-in data from Firestore.
2. Calculate the "Daily Workload" per team: Total task count (number of check-ins) and Total Distance traveled (sum of `distanceFromLastPoint`).
3. Display this data in a clean, modern summary table or cards using Tailwind CSS.
4. Add a simple date filter to let the admin view workload by specific days or months.
